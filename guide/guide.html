• Introduction
    • What is GEJ?
      GEJ is a simplest game engine available for beginners. It can be used to create games in the Java programming language. It has several pre-created actions, which are useful for beginners. This is also intended to be small in size and be event based just like the GameMaker.
    • Download and Installation
      You can always download the package here. GEJ is a free opensource game engine which is neatly packaged. Moreover it is very small in size too. After downloading, include it in your build path and your done installing it. You can also repack the .jar files into a single archive.
    • Package Details
      GEJ has three packages available for you to download. They are in the order here.
      > GEJ Binaries    -- The compiled engine
      > GEJ Source      -- The source code
      > GEJ Documentation   -- The JavaDocs
  • Starting to code with GEJ
    A game in GEJ must be the subclass of the Game class of the com.gej.core package. A perfect game would implement these methods. They are
    • initResources()
      In this method you can load the resources needed by the game like images, sounds etc.,
    • update(long elapsedTime)
      This method is used to control the game. It is called constantly all the time the game is running. The argument elapsedTime is time taken by the method in the last call.
    • render(Graphics2D g)
      This method is used to render the game to the screen. You could render background, the viewing part of the map, animated fonts etc.,

    The Basic Game skeleton

    Now Let's see a skeleton which is common for any GEJ game. // Java Foundation Classes (JFC)

        }

        public void render(Graphics2D g){

        }

  • Game States
    In GEJ, a game is a state machine. Each game has five states which are defined in the enum GameState of the com.gej.core package. They are
    • GAME_LOADING This states says that the game is loading resources either by the initResources() method or by the recommended ResourceManager class.
    • GAME_INTRO The game is currently showing the intro screen. Player has to see or press a key to start the game.
    • GAME_PLAYING This state says the the user is currently playing a level.
    • GAME_PAUSED Indicates that the game has been paused.
    • GAME_END The game is showing the end screen. Can be used to ask player whether he/she want's to restart or quit the game.
  • Resource Manager
    Every game needs images and sounds. To make loading them easily, GEJ provides a ResourceManager class. To make them easy, just make a new package called resources and make a class named Resources which contains names of the files. package resources;

    public class Resources {

        public static String IMAGE_1 = "resources/image1.png";

        public static String SOUND_1 = "resources/sound1.wav";

    }     Then you have to define them in the resource manager. So add this import. import static com.gej.core.ResourceManager.*; This makes you easily use the static methods of ResourceManager class. Then define the images and sounds in the initResources() method. java.awt.Image     IMG_1 = null;  // The image object
    com.gej.sound.WavSound SND_1 = null;  // The sound object

    public void initResources(){
        defineImage(Resources.IMAGE_1);
        defineSound(Resources.SOUND_1);
        loadResources(); // Start loading resources
    }     We've said to start loading resources. But if we start the gameplay with the null values, you'll get exceptions in the runtime and the game may not respond. To prevent this, you have to map your resources in the update() method. public void update(long elapsedTime){
        switch (getState()){
            // The game is a state machine
            case GAME_LOADING:
                if (!isLoading()){
                // Method of ResourceManager
                IMG_1 = getImage(Resources.IMAGE_1);
                SND_1 = getSound(Resources.SOUND_1);
                // Mapping completed. Change game state
                }
                break;
      }
    }

          }

      }

    • Updating the Object
      Now that you know how to use the update() method in the game. This method also exists in the GObject class and all that you need is override this method. The update() method of every object is called after the update method of the game if the game state is GameState.GAME_PLAYING. This method could be used to set the direction of the object and changing the image/animation according to the direction of the object and to check input. Let's see an example update() method. public void update(long elapsedTime){
          if (GKeyBoard.isPressed(KeyEvent.VK_LEFT)){
              setVelocityX(-4);
          } else if (GKeyBoard.isPressed(KeyEvent.VK_RIGHT)){
              setVelocityX(4);
          } else {
              // No movement key (needed by this object
              // is being pressed. So stop.
              setVelocityX(0);
          }
      }
    • Collision Detection
      The next most important component of a game is collision detection. To do the collision detection, we use the collision(GObject other) method. The argument other is the reference to the object which collided the current object. When two objects collide, a collision event is fired by the engine. To be specific, two collision events are fired, one for each object. public void collision(GObject other){
          if (other instanceof MyObject1){
              // The class name of the other is MyObject1
          } else if (other instanceof MyObject2){
              // other is MyObject2
          }
      }       A note to beginners who don't know concepts of game-programming is that we program only one class for each object and a single object can have multiple instances in the level.
    • Example Objects
      In this section, I wan't to show you an example which consists of two classes (two objects namely Object1 and Object2. Object1 moves to the right and when collides Object2, it reverses it's direction. public class Object1 extends GObject {

  • Maps and MapLoader
    • MapLoader interface
      Using the maps eases the level creation. To make levels, you need to implement the MapLoader interface from the com.gej.map package. At first we need a MapLoader. Here MapLoader refers to a class which implements the MapLoader interface. The common practice is to make the main game class implement this interface. Here's an example. public class MyMaps implements MapLoader {

          public GObject getObject(char ch, int x, int y){
              switch(ch){
                  case '1': return new Object1(x, y); break;
                  case '2': return new Object2(x, y); break;
              }
              return null;
          }

      }       See that we have used characters to denote objects and add them to the map. And there is also another method which you should implement. public Tile getTile(char ch, int x, int y){
          return null;
      }       You may notice that why this method is null. This is kept only for backwards compatibility and may be removed in the future versions.
    • MapInfo
      GEJ's Map class is a final static class. So it cannot be instantiated and it has only one instance. The same map is cleared and loaded every level. A map is a place where the objects live. The positions of the objects are described in the MapInfo in the form of strings. Let's see an example MapInfo. public class Map1 extends MapInfo {

          {
              // Set the tile size of the map for this level
              tileSize = 32;

              // The level data. Here are the conventions
              /**
               * Use the characters which are defined in the
               * MapLoader. If the character isn't defined,
               * it is considered as an empty square. Every
               * ' ' space is considered as an empty square.
               * Use '\n' to go to next row.
               *
               * In this example,
               *    '1' - Object 1
               *    '2' - Object 2
               */
              MapData = "\n"+
                        "1          2\n"+
                        "\n";
          }

          public Map1(MapLoader loader){
              setMapLoader(loader); // Set the maploader
          }

      }

      Drawing a map is also easy. It doesn't deal with which MapInfo is currently loaded. It is so simple to call the renderMap() method from the render() method. public void render(Graphics2D g){

      }
    • Map View
      Now that this method of creating maps is easier a lot, we have to think what if our map is larger and only a part of it is displayed? Here comes the MapView class. This class gives the effect of scrolling the map by simply changing the coordinates of the map. Map view translates these coordinates when it's static follow() method is called. So only one object should be followed. What you do is simply call the follow() method in the update() method and render using the MapView class. // The update() method in the object
      public void update(long elapsedTime){
          MapView.follow(this);
          // Other update code
      }

      // The render() method in the game
      public void render(Graphics2D g){
          MapView.render(g);
      }
    • MapManager
      To simplify the process of parsing and adding the objects to the Map, There exists another class called MapManager. This class contains an ordered list of MapInfo objects and provides an easier mechanism of switching the maps. It is so fast that parsing the MapInfo objects takes less than 10 milliseconds. This is how you use the manager. /** NOTE: This code should be placed
       *        in the game class which implements
       *        The MapLoader interface
       */

      // add the map to the MapManager
      MapManager.addMap(new Map1(this));
      MapManager.addMap(new Map2(this));

      // Starting the first map
      MapManager.startFirstMap();

      // Loading next map
      MapManager.loadNextMap();

      // Loading the previous map
      MapManager.loadPreviousMap();

      if (MapManager.hasNextMap()){
          // Do if there is a next map
      }

      if (MapManager.hasPreviousMap()){
          // Do if there is a previous map
      }
  • An example game
    In this section, we will be making a maze game. We create the images using the ImageTool class. The code should be placed in SimpleMaze.java // AWT imports

    import java.awt.event.*;

    // GEJ imports
    import com.gej.core.*;
    import com.gej.graphics.*;
    import com.gej.input.*;
    import com.gej.map.*;
    import com.gej.object.*;
    import com.gej.util.*;

    // Main game class
    public class SimpleMaze extends Game implements MapLoader {

        public static Image PLAYER;
        public static Image WALL;
        public static Image GOAL;

        public void initResources(){
            Global.TITLE = "A Simple Maze Game";
            PLAYER = ImageTool.getColoredImage(Color.RED,    32, 32);
            WALL   = ImageTool.getColoredImage(Color.GRAY,   32, 32);
            GOAL   = ImageTool.getColoredImage(Color.GREEN,  32, 32);
            Background.setBackground(ImageTool.getColoredImage(Color.BLACK,
                                                               Global.WIDTH,
                                                               Global.HEIGHT));
            MapManager.addMap(new MazeMap(this));
            MapManager.startFirstMap();
            setState(GameState.GAME_PLAYING);
        }

        public void update(long elapsedTime){
            if (GKeyBoard.isPressed(KeyEvent.VK_ESCAPE)){
                end_game();
            }
        }

        public void render(Graphics2D g){
            MapView.render(g);
        }

        public GObject getObject(char c, int x, int y){
            switch (c){
                case 'W': return new Wall(x, y);
                case 'P': return new Player(x, y);
                case 'G': return new Goal(x, y);
            }
            return null;
        }

        public Tile getTile(char c, int x, int y){
            return null;
        }

        public static void main(String[] args){
            GWindow.setup(new SimpleMaze());
        }

        public class Wall extends GObject {

            public Wall(float x, float y){
                super(SimpleMaze.WALL, x, y);
                setSolid(true);
            }

        }

        public class Goal extends GObject {

            public Goal(float x, float y){
                super(SimpleMaze.GOAL, x, y);
            }

        }

        public class Player extends GObject {

            final float SPEED = 4f;

            public Player(float x, float y){
                super(SimpleMaze.PLAYER, x, y);
            }

            public void update(long elapsedTime){
                MapView.follow(this);
                setVelocityX(0);
                setVelocityY(0);
                if (GKeyBoard.isPressed(KeyEvent.VK_LEFT)){
                    setVelocityX(-SPEED);
                } else if (GKeyBoard.isPressed(KeyEvent.VK_RIGHT)){

                }
                if (GKeyBoard.isPressed(KeyEvent.VK_UP)){
                    setVelocityY(-SPEED);
                } else if (GKeyBoard.isPressed(KeyEvent.VK_DOWN)){
                    setVelocityY(SPEED);
                }
            }

            public void collision(GObject other){
                if (other instanceof Goal){
                    // Start over again
                    MapManager.startFirstMap();
                }
            }

        }
